Для использования необходимы: Docker, Make, все остальное само устанавливается внутри Docker контейнера.


В Makefile можно найти все доступные команды, сделанные для простоты запуска кода, вот как их использовать:

## Инициализация

Обязательно нужно инициализировать БД в запущеном приложении при помощи make migrations и make migrate
```bash
make build
make up
make migrations
make migrate
```


# Остановка

Для остановки приложения используйте команду
```bash
make down
```


## Использование

Эндпоинт доступен по адресу POST http://localhost:8000/api/orders/ (он тестируется в тестах, если просто кинуть запрос на этот эндпоинт, скорее всего будет ошибка из за того, что нет нужного юзера и товара)

// Пример тела запроса
{
  "user_id": 1,
  "goods": [ { "good_id": 1, "quantity": 2 } ],
   // Поле promo_code может отсутствовать.
  "promo_code": "SUMMER2025"
}
// Пример тела ответа
{
    "user_id": 1,
    "order_id": 1,
    "goods": [ { "good_id": 1, "quantity": 2, "price": 100, "discount": "0.1", "total": 180 } ],
    "price": 200,
    "discount": "0.1",
    "total": 180,
}


## Тесты

```bash
make up
make test
```

**test_success** — проверяет успешное создание заказа без промокода, а также корректность расчета базовой цены, итоговой суммы и состава товаров.
**test_with_promo** — проверяет успешное применение валидного промокода, корректный расчет скидки на товар и итоговую сумму, а также привязку промокода к заказу.
**test_with_excluded_product** — проверяет, что промокод применяется только к обычным товарам, а для товаров, исключенных из скидок, скидка не начисляется.
**test_wrong_category** — проверяет, что промокод не применяется к товарам из категорий, на которые его действие не распространяется.
**test_promo_not_found** — проверяет, что при передаче несуществующего промокода возвращается код статуса 400 с ошибкой для поля `promo_code`.
**test_promo_expired** — проверяет, что использование просроченного промокода возвращает код статуса 400 с ошибкой для поля `promo_code`.
**test_promo_limit_reached** — проверяет, что при исчерпании общего лимита использований промокода возвращается код статуса 400 и ошибка для поля `promo_code`.
**test_promo_already_used** — проверяет, что повторное использование промокода тем же пользователем возвращает код статуса 400.
**test_user_not_found** — проверяет, что при передаче несуществующего идентификатора пользователя возвращается код статуса 400 с ошибкой для поля `user_id`.
**test_product_not_found** — проверяет, что при передаче несуществующего идентификатора товара в составе заказа возвращается код статуса 400.
**test_duplicate_products** — проверяет, что наличие дублирующихся товаров в списке заказа (одинаковые `good_id`) приводит к ошибке 400.
